Chapter II – Review of Related Literature and Systems

Section Comments Recommendations
1. Related Literature (Foreign & Local) The literature review includes several foreign studies on online counseling, teletherapy platforms, and mental health applications, but the local literature is sparse. The transition from global to local context is abrupt. Some cited works are from blogs or non-academic sources. Add at least 5 local studies from peer-reviewed journals, CHED repositories, or DOST/DepEd research archives to strengthen Philippine context. Arrange literature in thematic order (e.g., privacy in counseling systems, AI/chatbot use, user adoption in schools). Avoid blog-based citations unless backed by primary sources. Include synthesis tables summarizing each study’s scope, methods, findings, and relation to the current project.
2. Related Systems Several foreign and local systems are described (e.g., BetterHelp, e-Konsulta, university portals), but the analysis is descriptive rather than comparative. The unique features and shortcomings compared to the proposed system are not deeply analyzed. For each related system, add a feature-by-feature comparison matrix highlighting gaps that your system will address (e.g., real-time video sessions, crisis response protocol, offline access). Clearly explain how your system is an improvement or adaptation. Include screenshots of publicly available platforms with proper citation for visual reference.
3. Synthesis The synthesis currently reads as a simple summary of the literature and systems. It does not clearly show the gap that justifies the project. Develop a visual gap analysis chart showing: (a) what existing studies/systems cover, (b) what they lack, and (c) where your system fits. Explicitly connect the gap to your project’s objectives and problem statement. Use citations to back up the gap claims.

Chapter III – Methodology

Section Comments Recommendations
1. Research Design Stated as “Developmental” but without specifying the software development methodology (e.g., Agile, Rapid Application Development). The justification for choosing the approach is missing. Explicitly state the SDLC or Agile methodology to be used (e.g., Agile Scrum) and justify it based on the project’s nature (e.g., iterative feedback from guidance counselors and students). Provide a brief overview of each phase and its deliverables.
2. Development Tools Tools are listed (e.g., programming language, database) but without versions, hosting plan, or justification for their selection. Create a Development Tools Table with: Tool name, version, purpose, reason for selection, and possible alternatives. Mention hosting/server details (on-premises vs. cloud).
3. Requirements Specification Functional and non-functional requirements are not clearly separated. Accessibility and security requirements are not fully covered. Separate Functional and Non-Functional Requirements into tables. Include privacy compliance requirements (Data Privacy Act of 2012, RA 11036 Mental Health Act). Specify accessibility requirements (WCAG 2.1).
4. Architectural Design A general block diagram is mentioned but not shown. Data flow between modules is not detailed. Include: (1) System Architecture Diagram (client–server/cloud setup), (2) Data Flow Diagram (DFD) Level 0 and 1, and (3) ERD for the database. Clearly identify role-based access and how each module interacts.
5. Development Plan / Timeline Gantt chart included but lacks milestones for testing phases and user training. Update the Gantt chart to include unit testing, integration testing, UAT, security audit, and deployment. Assign responsible persons for each task.
6. Testing Plan Testing methods are briefly mentioned (e.g., functionality testing) but no metrics or tools are specified. Detail the testing plan: test cases, success criteria, tools (e.g., Selenium for UI testing), and performance benchmarks. Include usability testing with guidance counselors and students as part of UAT.
7. Implementation & Maintenance Plan Implementation is mentioned briefly; no details on maintenance. Add Deployment Plan (pilot test, full rollout, training) and Maintenance Plan (bug fixes, updates, backups). Indicate responsible team members and timeline for post-deployment support.
8. Ethical Considerations Data privacy and consent are mentioned but not detailed. Include informed consent procedures for all participants, storage encryption methods, data retention policy, and compliance with RA 10173 and RA 11036. Describe how minors’ consent will be handled (parental consent forms).

Overall Observations:

  • Chapter II needs more scholarly and local sources, better comparative analysis, and gap justification.

  • Chapter III needs clearer methodology structure, visual diagrams, testing details, and ethical compliance documentation.
