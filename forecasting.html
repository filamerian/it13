Proposed System Analysis

Automated Financial Forecasting and Operational Budgeting Dashboard for Short-Term Rental Businesses

Overall Evaluation

This is a highly ambitious and technically strong capstone proposal. It shows advanced thinking in data engineering, financial analytics, and system integration.

Compared to typical BSIT systems, this is above-average in concept quality and technical direction.

However, the current version is over-scoped and over-dependent on external systems (APIs and forecasting claims).

With proper refinement, this can become a very strong and defensible capstone project, but only if it is simplified and made more realistic.

Strengths

1. Strong Real-World Problem

The proposal correctly identifies a real issue:

  • fragmented financial data across platforms
  • manual spreadsheet consolidation
  • difficulty determining true profitability
  • lack of centralized reporting

This is a valid business information system problem.

2. High BSIT Value

The system involves:

  • database design and management
  • data integration concepts (ETL-style thinking)
  • financial computation logic
  • dashboard development
  • analytics and reporting

These are strong BSIT-aligned competencies.

3. Clear Target Users

Well-defined users:

  • property managers
  • short-term rental hosts
  • investors

This supports meaningful system evaluation.

4. Good System Direction

Even if simplified, the system can provide:

  • centralized financial visibility
  • operational monitoring
  • performance tracking per property

This is a strong foundation for a capstone system.

Areas for Improvement (Key Refinements Needed)

1. Reduce Dependence on External APIs

Current Risk:

  • Airbnb / VRBO API integration may not be accessible
  • banking APIs are restricted
  • scraping is legally and technically risky

Recommendation:

“Manual import of financial data (CSV upload or encoded transactions)”

OR

“Simulated API dataset for system demonstration”

This makes the system fully controllable and defendable.

2. Reframe “ETL Pipeline” into a Simpler Concept

Current:

Automated ETL from multiple platforms

Recommended:

“Data Import and Financial Consolidation Module”

Focus on:

  • uploading data
  • cleaning/formatting
  • storing unified records

This keeps the idea but removes heavy infrastructure dependency.

3. Simplify Forecasting Feature

Current Risk:

  • unclear predictive model
  • too AI-heavy
  • difficult to defend mathematically

Recommendation:

“Historical Trend-Based Financial Projection”

Examples:

  • average monthly revenue projection
  • seasonal comparison charts
  • simple moving average trends

This is defensible and implementable.

4. Reduce “Enterprise-Level Language”

Avoid terms like:

  • “C-Suite dashboard”
  • “financial intelligence ecosystem”
  • “debt optimization engine”

Replace with:

  • “admin dashboard”
  • “financial summary dashboard”
  • “loan tracking module”

This makes it more academically appropriate.

5. Define a Clear MVP (Minimum Viable Product)

The system should clearly focus on:

Core System Scope:

  • property management
  • income and expense tracking
  • profit computation per property
  • monthly reporting
  • dashboard visualization

Everything else should be optional enhancements.

Recommended System Structure (Refined Version)

Core Modules

  • User Management (Owners / Managers)
  • Property Listing Management
  • Income & Expense Tracking
  • Profit Computation Engine
  • Financial Dashboard
  • Monthly Reports Generator

Optional Enhancements

  • Data Import (CSV upload)
  • Trend Charts (historical only)
  • Simple forecasting (average-based projection)

Remove or Defer

  • direct Airbnb/VRBO API integration
  • advanced predictive AI models
  • debt optimization algorithms
  • multi-platform scraping systems

Revised Problem Statement Focus

Instead of emphasizing “AI forecasting,” focus on:

  • fragmented financial records
  • manual consolidation of income/expenses
  • difficulty tracking profitability per property
  • lack of centralized reporting system

This is much more defensible in a capstone defense.

Revised Objectives (Recommended)

General Objective

To design and develop a financial management and reporting system for short-term rental businesses that centralizes income, expenses, and performance monitoring.

Specific Objectives

  1. To develop a system for recording property income and operational expenses.
  2. To compute net profit per property automatically.
  3. To generate monthly financial reports for property owners and managers.
  4. To provide a dashboard for visualizing financial performance trends.
  5. To allow data import and consolidation from external records.

Final Recommendation

APPROVED WITH MAJOR SCOPE REFINEMENT


This is a strong capstone idea if simplified properly.

The success of this project depends on:

  • removing external API dependency
  • simplifying forecasting to trend analysis
  • focusing on financial recording + reporting
  • keeping the system implementable within BSIT timeline

Final Coordinator Insight

This proposal has high potential but is currently “over-engineered in concept.”

Once refined, it becomes:

a very strong financial management system with analytics capability

instead of:

a risky enterprise-level forecasting platform
