Proposed System Analysis

Emergency Response System Platform for Jamindan

Overall Evaluation

This proposal addresses a legitimate and socially important operational problem involving emergency incident reporting and response coordination within the Municipality of Jamindan.

Compared to many generic monitoring systems, this proposal has stronger real-world relevance because it focuses on disaster response, emergency communication, and incident coordination among multiple agencies.

However, while the concept is strong, the current proposal is still over-scoped and contains several features that may become difficult to complete within the capstone timeline.

The project has strong potential if the proponents focus only on core emergency reporting and coordination functionalities.

Strengths

1. Real and Relevant Community Problem

The proposal addresses real operational issues during emergencies such as:

  • delayed emergency response
  • incomplete incident information
  • difficulty identifying exact locations
  • uncoordinated communication channels

These are legitimate problems during:

  • typhoons
  • flooding
  • fires
  • medical emergencies
  • road accidents

2. Strong Public Service Value

The system directly benefits:

  • residents
  • emergency responders
  • barangay officials
  • municipal disaster offices

This gives the project high community impact and practical relevance.

3. Strong BSIT Alignment

The proposal includes:

  • database management
  • GPS integration
  • notification systems
  • dashboard development
  • report management
  • communication modules

These are strong BSIT-related competencies.

4. Existing Development Progress

The group already completed:

  • UI design
  • dashboard design
  • database planning
  • system architecture planning

This is a positive indicator because development has already started.

Major Concerns

1. Scope is Too Large

The proposal currently attempts to include:

  • incident reporting
  • GPS tracking
  • alert systems
  • communication modules
  • news and updates
  • safety guidelines
  • dashboard analytics
  • activity logging
  • APK deployment

This is already a large-scale information system.

The project risks becoming too broad for a BSIT capstone if all features are fully implemented.

2. Communication Module Complexity

The communication module may significantly increase system complexity.

A panel may ask:

Is this an emergency reporting system or a real-time communication platform?

Real-time messaging introduces:

  • synchronization issues
  • notification management
  • server load concerns
  • security considerations

This may become difficult to complete properly within the timeline.

3. WebView APK is Weak as a Mobile Solution

The proposal mentions:

WebView APK for Mobile Access

Panels may challenge this because:

  • it is essentially a wrapped website
  • it may not function fully offline
  • native emergency functionality may be limited

The proponents should clarify whether:

  • the system is primarily web-based
  • or a true mobile emergency application

4. GPS Accuracy and Emergency Reliability

The system heavily depends on GPS location tracking.

Possible concerns include:

  • weak mobile signal during disasters
  • inaccurate GPS coordinates
  • user inability to access internet connection

The proposal should clearly define limitations and fallback procedures.

5. Too Many Stakeholders

Current stakeholders include:

  • LGU
  • MDRRMO
  • PNP
  • BFP
  • Volunteers
  • Barangay Officials

This may create workflow complexity.

For a capstone prototype, the project should prioritize:

  • Residents
  • MDRRMO
  • Barangay Officials

Additional agencies can be future enhancements.

Recommended Scope Reduction

Keep

  • User Registration and Login
  • Incident Reporting
  • GPS Location Detection
  • Emergency Notifications
  • Command Center Dashboard
  • Report Status Tracking
  • User Management

Remove or Simplify

  • Advanced analytics
  • Complex communication modules
  • News and updates module
  • Emergency tips module
  • Complex activity logs
  • Multi-agency workflow automation

Recommended Problem Statements

  1. Emergency incidents in Jamindan are commonly reported through phone calls, text messages, and social media, causing delays and incomplete information.
  2. Emergency responders have difficulty identifying the exact location of incidents during emergencies.
  3. There is no centralized platform for monitoring and managing emergency reports within the municipality.
  4. Manual coordination between residents and emergency response agencies may result in delayed response times.
  5. Local emergency offices lack a digital system for tracking incident reports and response activities.

Recommended Objectives

General Objective

To design and develop an Emergency Response System Platform that improves incident reporting and emergency coordination in the Municipality of Jamindan.

Specific Objectives

  1. To develop a digital incident reporting system with photo upload and GPS location detection.
  2. To implement a centralized dashboard for monitoring emergency incidents.
  3. To provide emergency notifications and status updates for reported incidents.
  4. To create a database for storing emergency reports and activity records.
  5. To improve coordination between residents and emergency response personnel through a centralized platform.

Coordinator Verdict

Status: Needs Major Revision


Reasons

  • Strong and socially relevant concept.
  • Good BSIT alignment and practical value.
  • Current scope is too broad for the timeline.
  • Communication and multi-agency modules may become too complex.
  • Critical backend and integration components are not yet started.

Final Recommendations

  • Focus on core emergency reporting and GPS tracking first.
  • Simplify the communication module.
  • Limit the number of stakeholder workflows during initial implementation.
  • Prioritize backend development and GPS integration immediately.
  • Keep the system lightweight and usable for elderly and non-technical users.

If the proponents reduce the scope and focus on essential emergency response functionalities, this project can become a strong and impactful BSIT capstone with practical deployment potential for the Municipality of Jamindan.
