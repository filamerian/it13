Capstone Review: DINEBRIX

General Evaluation

The proposed system DINEBRIX is acceptable as a BSIT capstone project as per CHED and CCS standards. It addresses a real problem in restaurant inventory and pricing management. The system is now stronger because it includes QR-code tracking and a sales/payment recording module.

Overall, the study is feasible, relevant, and appropriate for BSIT level, but some improvements are still needed to make it more complete and defensible.

Positive Comments

  • The system solves a real problem in steakhouse inventory management.
  • The use of gram-based tracking improves accuracy in pricing and stock control.
  • The QR-code feature adds automation and reduces manual work.
  • The sales and payment recording module improves business monitoring.
  • The scope is realistic for a student capstone project.

Alternative Titles

  • DINEBRIX: A Web-Based QR-Code Enabled Inventory and Sales Recording System for Steakhouse Management
  • DINEBRIX: A Web-Based Inventory, Pricing, and Sales Recording System with QR-Code Tracking for Steakhouse Operations
  • DINEBRIX: An Integrated Web-Based System for Inventory Control and Sales Recording Using QR-Code Technology in Steakhouse Operations

Key Recommendations

1. QR-Code Inventory Tracking (Recommended Improvement)

Each steak item should have a QR code.

Staff can scan the QR code when steak is used or added.

The system will automatically update the remaining grams.

Benefit: Faster and more accurate inventory updates with less manual encoding.

2. Payment Module (Recording Only - Important)

The system should only record payments, not process online payments.

It should support:

  • Cash
  • GCash
  • Card (for recording only)

The system calculates totals and change automatically.

Important Reminder: Do NOT include real online payment integration (like banking or API payments). It is only for recording transactions.

3. Improve Theoretical Framework

Add another theory like DeLone & McLean IS Success Model.

This will help measure system success (accuracy, usability, user satisfaction).

4. Improve Objectives

Objectives should focus on results, not just system features.

Example: focus on improving accuracy, speed, and efficiency.

5. Scope Reminder

The system should only include:

  • Inventory tracking (grams)
  • QR scanning
  • Sales and payment recording
  • Reports

It should NOT include:

  • Online ordering
  • Delivery system
  • Real payment processing

Final Remarks

The system is already good for a BSIT capstone project. With QR-code tracking and sales recording added, it becomes more complete and practical. However, you should ensure that the payment feature is only for recording and not for real transactions.
