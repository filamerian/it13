Proposed System Analysis

IoT-Based Secure Coin Changer System for Lukas Gaming Center

Overall Evaluation

This is a practical and industry-oriented BSIT capstone project because it solves a real operational problem in a specific business environment.

Unlike many proposed systems that only digitize records, this project combines hardware, software, IoT, automation, and security, making it more technically substantial.

However, the proposal currently has several weaknesses that need to be addressed before it can be considered ready for implementation.

Strengths

1. Real and Observable Problem

The problem exists in actual operations.

Example:

  • Customer wants to exchange bills for coins.
  • Owner is unavailable.
  • Customer cannot immediately use the pisonet machines.
  • Business loses potential revenue.

This is a legitimate operational problem.

2. Clear Users

The beneficiaries are clearly identified:

  • Gaming center owner
  • Customers

This is better than projects with vague stakeholders.

3. Strong BSIT Components

The proposal includes:

  • Embedded systems
  • IoT
  • Database
  • Web monitoring
  • Security mechanisms
  • Transaction logging

These align well with BSIT outcomes.

Major Weaknesses

1. Problem Statement Needs More Evidence

Current problem:

Owner is not present to exchange money.

This is too narrow.

A panel may ask:

"How often does this actually happen?"
"Does it happen enough to justify building an entire automated machine?"

The group needs actual data such as:

  • average daily coin exchange requests
  • number of delayed transactions
  • customer complaints
  • owner interviews

Without evidence, the problem appears anecdotal.

2. Scope is Too Hardware-Heavy

The system includes:

  • bill validator
  • coin acceptor
  • coin dispenser
  • ESP32
  • Firebase
  • web dashboard
  • motion sensor
  • GSM alerts

This is a large hardware integration project.

The biggest risk is not coding.

The biggest risk is:

  • mechanical reliability
  • dispensing accuracy
  • sensor integration
  • hardware procurement

Many capstone groups underestimate this.

3. Security Claims are Too Broad

The proposal mentions:

Anti-Theft and Security System

This is vague.

Questions likely asked by panels:

  • What theft scenarios?
  • How is theft detected?
  • What happens when theft is detected?
  • How reliable is detection?

Security must be clearly defined.

4. No Working Prototype Yet

Current status:

None of the system components are currently working.

This is concerning.

Most successful hardware capstones already have:

  • ESP32 tests
  • Firebase connection
  • sensor prototypes

before proposal approval.

Novelty Assessment

Is it Novel?

Partially.

Coin changers already exist commercially.

Weak Novelty Argument

"It changes bills into coins."

Not novel.

Strong Novelty Argument

The novelty should focus on:

  • low-cost implementation for small gaming centers
  • IoT monitoring capability
  • remote inventory monitoring
  • theft detection alerts
  • transaction history and analytics

The novelty is not the changer itself.

The novelty is the integration of:

Automated coin changing + IoT monitoring + security tracking for small gaming businesses.

Recommended Problem Statements

  1. Lukas Gaming Center relies on manual coin-changing transactions that require the owner's presence, causing delays in customer transactions.
  2. There is no automated mechanism for converting bills into coins for pisonet users.
  3. The owner has no real-time monitoring tool to track machine usage, coin inventory, and transaction records remotely.
  4. Coin inventory levels are manually monitored, increasing the risk of service interruption when coins run out.
  5. There is no automated alert system to notify the owner of possible tampering, unauthorized access, or machine-related issues.

Recommended Objectives

General Objective

To design and develop an IoT-Based Secure Coin Changer System for Lukas Gaming Center that automates coin-changing transactions and provides real-time monitoring and security features.

Specific Objectives

  1. To develop an automated bill acceptance and coin dispensing mechanism.
  2. To implement a currency validation system for accurate transaction processing.
  3. To develop a web-based monitoring dashboard for viewing transactions and machine status.
  4. To integrate an IoT platform that records transaction data in real time.
  5. To implement a security alert system that notifies the owner of potential tampering or abnormal activities.

Scope Reduction Recommendation

For the 50% milestone, focus only on:

Core Features

  • Bill acceptance
  • Coin dispensing
  • ESP32 integration
  • Firebase logging
  • Web dashboard
  • Basic security alert

Move to Future Enhancement

  • Advanced anti-theft mechanisms
  • Analytics
  • Predictive maintenance
  • Multi-machine monitoring

Coordinator Verdict

Status: Approved with Major Revision


Reasons:

  • The problem is real and appropriate.
  • The system demonstrates strong BSIT competencies.
  • The novelty should be reframed around IoT monitoring and automation, not coin changing alone.
  • The proposal lacks evidence that the problem occurs frequently enough.
  • Hardware integration risk is extremely high and should be prioritized immediately.

My biggest concern is not the concept itself, but the fact that none of the hardware or software components have been started yet.


For a hardware-based capstone, this group should begin prototyping immediately, because implementation and troubleshooting will consume far more time than expected.
