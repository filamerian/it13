WEB-BASED EVENT MANAGEMENT SYSTEM FOR VEVENTS PHOTOGRAPHY

CHAPTER I – INTRODUCTION

I. Overview of the Current State of Technology
Strengths:

  • Highlights relevant technologies in the event planning industry, including mobile apps, social media integration, and data analytics.
  • Mentions how technology improves efficiency, personalization, and evaluation in event planning.

Suggestions for Improvement:

  • The overview is general; consider anchoring the discussion with real-life use cases or platforms in the photography or event planning sector (e.g., Canva for event design, TagVenue for bookings).
  • Add references to support trends mentioned, such as academic or industry studies that show technology’s role in customer engagement and ROI in event management.
  • Reframe sentences for scholarly tone—avoid repetitive phrasing like “Technology is transforming…” and aim for synthesis.

Recommended Enhancements:
Include local context or Filipino event platforms, and transition more clearly into the specific challenges faced in photography-centered event planning.

II. Desired State of Technology
Strengths:

  • Clearly outlines the proposed solution—a web platform for venue booking, photography packages, and interactive event customization.
  • Shows intention to streamline booking, communication, and personalized planning.

Suggestions for Improvement:

  • Define the user personas: Are customers primarily wedding clients, corporate organizers, or hobbyists?
  • Clarify the “package deal” customization process—Is it template-driven or dynamic?
  • Technical details about database, notifications, or mobile responsiveness could reinforce feasibility.

Recommended Enhancements:
Include a high-level feature map and describe core innovations that differentiate the platform from existing competitors.

III. Statement of the Problem
Strengths:

  • Enumerates specific challenges: navigation issues, lack of flexibility, integration gaps, and brand differentiation problems.
  • Addresses both user experience and backend limitations.

Suggestions for Improvement:

  • Convert bullet points into formal research questions to guide methodology. For example:
    • How intuitive is the navigation experience for first-time users?
    • What integrations significantly improve platform functionality and user engagement?
  • Add a short transition paragraph linking user frustrations to technological limitations and the research gap.

Recommended Enhancements:
Provide examples of current systems that fail to meet these expectations to highlight relevance and need.

IV. Objectives of the Study
Strengths:

  • General objective clearly aligns with the central goal of developing an intuitive web-based event organizer.
  • Specific objectives address interface, customization, integration, data security, and brand development.

Suggestions for Improvement:

  • Formalize phrasing in specific objectives: Use “To improve…” or “To develop…” consistently.
  • Add measurable outcomes: For instance, “increase user retention by 20%” or “reduce booking steps to 3 clicks.”

Recommended Enhancements:
Group objectives into technical (UI, security), functional (customization, booking), and strategic (branding, marketing) clusters to show breadth.

V. Theoretical Framework
Strengths:

  • Applies the Task-Technology Fit (TTF) theory, appropriate for systems aiming to enhance performance by aligning tech features with user needs.
  • Identifies core elements: Technology characteristics, task characteristics, performance impact, and utilization.

Suggestions for Improvement:

  • Framework text needs refinement; several descriptions are duplicated or vague (“easy viewing of services,” “responsive and reliable”).
  • Enhance academic tone by explaining how each TTF element directly influences platform success metrics (e.g., task complexity vs. UI design).

Recommended Enhancements:
Include scholarly citations for TTF and consider adding complementary frameworks like TAM (Technology Acceptance Model) for user adoption insights.

VI. Conceptual Framework
Strengths:

  • Identifies key modules: booking, multimedia uploads, calendar, admin dashboard, login security.
  • Describes CRUD operations and customizations for photographers.

Suggestions for Improvement:

  • Translate diagram references into a structured system flow or UML to show interactions between users, modules, and database.
  • Explain how “customizing packages” reflects user feedback or adaptive pricing features.

Recommended Enhancements:
Replace image placeholders with academically formatted diagrams and explicitly define relationships in the architecture.

VII. Operational Definition of Terms
Strengths:

  • Includes essential concepts such as Web-Based System, Event Booking, and Photographer.
  • Defines terms relevant to user interface and platform roles.

Suggestions for Improvement:

  • Add citations or references for definitions—e.g., IEEE glossary, Techopedia, or scholarly dictionaries.
  • Remove redundancy (e.g., “Styles” overlaps with booking customization).

Recommended Enhancements:
Turn this into a glossary table for clarity and reference usability throughout the proposal.

VIII. Scope and Limitations
Strengths:

  • Addresses platform constraints, functionality range, and user experience goals.
  • Mentions security, accessibility, and branding scope.

Suggestions for Improvement:

  • Structure into two distinct lists (Scope vs. Limitations).
  • Clarify which services are excluded (e.g., mobile app, third-party payment gateway integration).
  • Include assumptions (e.g., stable internet, browser compatibility).

Recommended Enhancements:
Define project duration, user types, geographic reach, and platform boundaries in a concise paragraph or table.

IX. Significance of the Study
Strengths:

  • Highlights skills development for students and improvements for event clients.
  • Emphasizes coordination, vision translation, and user engagement.

Suggestions for Improvement:

  • Reframe informal phrases (“whole host of transferable skills”) into structured academic prose.
  • Expand on institutional benefits (e.g., faster vendor coordination, reduced overhead) and stakeholder analysis.

Recommended Enhancements:
Add alignment with professional standards (e.g., ISO usability requirements, industry expectations for freelance photographers).
