Proposed System Analysis

Arduino-Powered Automated Bar Gate with Proximity Detection in St. Joseph Village

Overall Evaluation

In its current form, this proposal is one of the weaker capstone proposals reviewed so far. The idea is not inherently bad, but the group has not clearly identified a significant problem, validated its existence, or justified why an automated gate system is needed.

The proposal currently appears to be a technology-first project rather than a problem-driven capstone project.

A BSIT capstone should start with a real operational problem and then propose a technology solution. In this case, the group appears to have started with the idea of an automated gate and then tried to find a problem afterward.

Major Concerns

1. Problem Validation is Weak

The group answered:

"Not sure"

when asked whether the problem exists in real life.

This is a serious issue.

If the proponents themselves are unsure whether the problem exists, then the foundation of the capstone is weak.

A panel will likely ask:

  • How many unauthorized entry incidents occurred?
  • How often do guards fail to identify vehicles?
  • Are there security reports supporting this problem?

Currently, there is no evidence.

2. Problem Occurs "Rarely"

The group indicated:

Rarely

This creates another challenge.

A capstone should generally solve a recurring operational problem.

Why build an entire automated gate system for a problem that rarely happens?

The cost and complexity of the solution may not be justified by the frequency of the problem.

3. Unclear System Scope

The title includes:

  • Automated Bar Gate
  • Proximity Detection

The features include:

  • Car Identification
  • Automated Gate
  • Car Logs

The core feature is:

Plate Number Identification

These are different technologies.

The group must clearly define whether the system uses:

Option A

License Plate Recognition

or

Option B

RFID / proximity-based vehicle identification

Trying to implement both significantly increases complexity.

4. Novelty is Weak

Commercial systems already exist that provide:

  • automatic gates
  • RFID access control
  • license plate recognition
  • vehicle logging

The proposal currently does not explain what is new or improved.

"Why not just buy an existing gate system?"

The group currently has no strong answer.

5. Beneficiaries are Too Broad

Current users:

Home Owner / Residence

This is vague.

Who actually uses the system?

  • Security guards?
  • Homeowners?
  • Village administrators?

The primary user should be clearly identified.

Technical Risk Assessment

Positive

The web-based side is already working.

This suggests:

  • database connectivity
  • interface development
  • logging features

may already be progressing.

Concern

The entire hardware side remains unfinished.

This includes:

  • gate mechanism
  • sensors
  • controller integration
  • vehicle detection

For hardware projects, these are usually the most difficult parts.

How to Save the Project

The project becomes much stronger if it is reframed as:

Current Idea

Automated Gate System

Better Idea

Village Vehicle Access Monitoring and Automated Gate Control System

The emphasis shifts from merely opening a gate to:

  • access control
  • visitor tracking
  • security logging
  • vehicle monitoring

This creates a stronger BSIT justification.

Recommended Problem Statements

  1. St. Joseph Village currently relies on manual monitoring of vehicle entry and exit at the village gate.
  2. Security personnel have limited tools for verifying and recording vehicle access information.
  3. Vehicle entry logs are manually recorded, making monitoring and retrieval of records inefficient.
  4. There is no automated mechanism for controlling gate access based on authorized vehicle identification.
  5. Village administrators lack a centralized system for monitoring vehicle entry and exit activities.

Recommended Objectives

General Objective

To design and develop an Automated Vehicle Access Monitoring and Gate Control System for St. Joseph Village.

Specific Objectives

  1. To develop a vehicle identification mechanism using a selected access technology.
  2. To implement an automated gate control system using Arduino.
  3. To develop a database for storing vehicle entry and exit records.
  4. To create a web-based monitoring dashboard for authorized personnel.
  5. To generate vehicle access logs and monitoring reports.

Novelty Assessment

Current Novelty: 4/10

The system is very similar to existing commercial gate automation systems.


Improved Novelty: 7/10

If focused on:

  • localized village implementation
  • integrated vehicle logging
  • low-cost Arduino deployment
  • centralized monitoring

the project becomes more defensible.

Coordinator Verdict

Status: Needs Major Revision


Reasons

  • Problem validation is weak.
  • Problem frequency is reported as "rare."
  • Novelty is currently insufficient.
  • Technology selection is unclear.
  • The proposal appears solution-driven rather than problem-driven.

Recommendation

The group should first validate whether access control and vehicle monitoring are genuine concerns within St. Joseph Village. If evidence can be gathered, the project should be reframed as a vehicle access monitoring and security management system rather than simply an automated gate.


Without stronger problem justification, this proposal may struggle during proposal defense and final evaluation.
