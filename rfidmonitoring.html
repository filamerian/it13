Proposed System Analysis

Mobile-Based RFID Monitoring System for FCU Kindergarten Department

Overall Evaluation

This is a strong, relevant, and socially valuable capstone proposal focused on child safety, attendance automation, and guardian verification.

However, while the problem is valid and well-defined, the system currently contains overstated technical claims (especially “real-time location tracking”) and a scope that must be carefully controlled to remain feasible and ethical for BSIT implementation.

With refinement, this can become a very strong and defensible capstone system.

Strengths

1. Highly Relevant Real-World Problem

The proposal addresses a genuine school concern:

  • manual child pickup verification
  • delayed guardian identification
  • lack of real-time notification
  • safety risks during dismissal

This is a clear administrative and safety workflow problem, not theoretical.

2. Strong Practical Value

If implemented properly, the system can:

  • speed up dismissal process
  • improve guardian verification
  • reduce human error in child release
  • improve communication between school and parents

3. Clear Stakeholders

Well-defined users:

  • kindergarten teachers
  • school administrators
  • parents/guardians

This ensures easy validation, real deployment potential, and clear system roles.

4. Good BSIT Scope Fit (If Controlled)

The system involves:

  • RFID integration
  • mobile application development
  • database management
  • notification system
  • user authentication

These are valid BSIT-level competencies.

Major Issues and Required Refinements

1. “Real-Time Location Tracking” is Misleading

Current Claim: Real-time pupil location tracking

RFID systems do not provide continuous location tracking. They only detect entry/exit points and scan events.

Recommendation:

“Real-time attendance and pickup event tracking”

2. Risk of Over-Surveillance Perception

The system may be interpreted as a child tracking surveillance system, which is sensitive because it involves minors.

Recommendation:

Frame the system as a secure pickup verification and attendance monitoring system, not a tracking system.

3. Scope is Slightly Overloaded

Current features include:

  • RFID scanning
  • mobile apps for multiple users
  • notifications
  • reports
  • guardian verification
  • centralized database

This is acceptable but should remain carefully structured.

4. No Working Prototype Yet

The project currently has no working prototype, which increases risk because RFID hardware integration and mobile development require extensive testing.

The group should prioritize RFID scanning, login, and attendance logging immediately.

Recommended Scope Refinement

Core System (Must Have)

  • RFID-based child identification (scan-in / scan-out)
  • Guardian verification system
  • Attendance and pickup logging
  • Teacher dashboard
  • Parent notification system
  • Centralized database

Optional Enhancements

  • mobile app for parents
  • attendance reports
  • simple analytics

Remove / Avoid

  • real-time location tracking
  • continuous tracking claims
  • AI or predictive features

Recommended Problem Statements

  1. FCU Kindergarten Department relies on manual verification of guardians during pupil pickup, which may cause delays and identification errors.
  2. Teachers face difficulty in quickly confirming authorized guardians during dismissal hours.
  3. There is no centralized digital system for recording pupil attendance and pickup logs.
  4. Parents currently do not receive immediate confirmation when their child is picked up.
  5. Manual monitoring methods reduce efficiency and may pose safety risks for kindergarten pupils.

Recommended Objectives

General Objective

To design and develop an RFID-based mobile monitoring system that improves the security and efficiency of pupil pickup and attendance management in FCU Kindergarten Department.

Specific Objectives

  1. To develop an RFID-based identification system for pupils and authorized guardians.
  2. To implement a digital attendance and pickup logging system.
  3. To create a system for verifying authorized guardians during pupil release.
  4. To develop a notification system that informs parents of pickup events.
  5. To design a centralized database for storing attendance and pickup records.

Final Recommendation

APPROVED WITH MINOR REVISION

Conditions:

  • remove “real-time location tracking” claim
  • clarify RFID as event-based tracking only
  • ensure guardian verification is properly secured
  • focus on attendance and pickup workflow only
  • build early RFID prototype to reduce implementation risk

Final Coordinator Insight

This is a strong and socially meaningful capstone because it solves a real school safety problem.

Its success depends on keeping it simple, event-based, and process-focused — not surveillance-based or over-tracked.

If refined properly, this will be one of the most practical and deployable systems in the batch.
