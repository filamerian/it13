III. Statement of the Problem
Comments:
The section identifies five multi-layered challenges faced by CACAG. It highlights gaps in security, location precision, evidence reporting, and stakeholder coordination.
Suggestions:

  • Ensure all items are phrased as formal statements, not inferred questions.
  • Link each problem to direct consequences (e.g., 'Delayed reporting increases fire spread risk').
  • Consider listing community-level implications, such as risk exposure and resource strain.

Recommendations:
Refine the structure as follows:

  1. Security vulnerabilities and lack of verification protocols expose CACAG's system to false alarms and unaccountable reporting.
  2. Manual and delayed communication channels prolong emergency response times and complicate incident validation.
  3. Limited location and evidence documentation lead to misdirection and inefficient deployment of fire responders.
  4. Inadequate coordination systems prevent real-time updates and shared situational awareness among responders.
  5. Absence of access control and secure data management compromises user privacy, app reliability, and long-term sustainability.

IV. Objectives of the Study
Comments:
The general and specific objectives are well scoped and emphasize technical robustness. However, some entries blend system actions with implementation goals, resulting in overlap.
Suggestions:

  • Use action verbs consistently (e.g., 'Create,' 'Design,' 'Evaluate,' 'Implement').
  • Link each specific objective to one of the identified problem statements.
  • Introduce measurable targets for evaluation (e.g., 'Achieve 95% GPS accuracy,' 'Reduce average incident reporting time to under 60 seconds').

Recommendations:
General Objective:
To design and deploy a secure mobile-based emergency reporting system for CACAG that enables real-time fire alerts, location tracking, and authenticated user submissions, thereby improving response speed, data accuracy, and community safety.
Specific Objectives:

  1. Develop and validate age-restricted registration with government ID and phone verification.
  2. Integrate GPS and timestamp-based media capture to establish verifiable incident logs.
  3. Implement automated notification protocols for responders and community stakeholders.
  4. Design anti-abuse logic (rate limits, user scoring, device tracking) to reduce false reporting.
  5. Build and evaluate a one-tap alert mechanism with real-time validation and cloud-based media storage.
  6. Assess system performance based on accuracy, latency, security compliance, and user satisfaction.

VIII. Scope and Limitations

Suggestions:

  • Users: CACAG responders and registered citizens (18+).
