FIRESENSE: REAL-TIME FIRE DETECTION AND GPS-BASED EMERGENCY NOTIFICATION SYSTEM FOR CACAG

MODULES TO BE DEVELOPED/CHECKED

Core Module Objective(s) Addressed Detailed Functions
1. User Registration & Authentication Resolve verification and security issues (Objective 1) • Government ID upload and verification
• Age check (≥18 years)
• Mobile number validation via OTP
• Login/Logout with encrypted credentials
• Role management (user, responder, admin)
• Device fingerprinting for anti-abuse
2. One-Tap Emergency Alert Reduce delays in reporting (Objective 2) • Single-button emergency trigger
• Automatic sending of user profile and verified credentials
• Instant alert generation for responders
• Rate-limit protection to prevent spam
3. GPS Location Tracking & Mapping Address location challenges (Objective 3) • Automatic capture of GPS coordinates on alert
• Real-time map visualization of incident location for responders
• Route guidance using Google Maps or equivalent API
• Geo-tagging of photos/videos
• Location history log for analytics
4. Evidence Submission (Photo/Video) Strengthen verification and accountability (Objective 3) • Photo or video upload during report
• Automatic timestamp and geotag on media
• Secure cloud storage with encryption
• Preview and confirm evidence before submission
5. Automated Notification System Improve communication and coordination (Objective 4) • Push notifications for CACAG responders and stakeholders
• SMS gateway fallback for poor internet areas
• Incident status updates (received, verified, dispatched, resolved)
• Broadcast alerts to multiple responders simultaneously
6. Incident Management Dashboard (Admin/Responder) Coordinate response actions (Objective 4) • Real-time incident queue with priority flags
• Filter by location, type, or urgency
• View submitted evidence and GPS location
• Update status (en route, on-site, resolved)
• Generate reports for analysis and audit
7. Data Privacy & Security Module Ensure accountability and protect sensitive information (Objective 5) • Encrypted storage of personal data and evidence
• Role-based access control (only authorized personnel can view sensitive info)
• Compliance with local data privacy law (e.g., Philippines Data Privacy Act)
• Audit logs for all system activities
8. Anti-Abuse Mechanisms Reduce prank calls and false alarms (Objective 1 & 3) • Report rate limiting per user/device
• User reputation scoring (trusted/untrusted reports)
• Device ID and IP logging to detect repeat offenders
• Admin tools to suspend or flag abusive accounts
9. Accessibility & Multilingual Support Increase usability across community members (Objective 2) • Interface in English/Filipino (or local dialect)
• Large buttons, simple navigation for first-time users
• Optional voice prompts for low-literacy users
