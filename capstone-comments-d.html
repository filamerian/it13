APARTMENT MANAGEMENT SYSTEM FOR CASA ORO

CHAPTER I – INTRODUCTION

I. Overview of the Current State of Technology
Strengths:

  • Offers a highly informative survey of emerging technologies in property management, including IoT, AI, AR/VR, and blockchain.
  • Establishes the relevance of digital transformation in the Philippine real estate sector, including examples like mobile apps, smart devices, and cloud-based systems.
  • Includes technical depth suitable for a senior IT research project.

Suggestions for Improvement:

  • Split the section into two subsections: (a) Global Trends in AMS and (b) Philippine Context of AMS Adoption.
  • Use statistics or cited studies to support claims about adoption rates or system impact (e.g., “AMS adoption is rising in the Philippines…”).
  • Refine narrative flow to reduce repetition across paragraphs. Several sentences reintroduce AMS definitions unnecessarily.

Recommended Enhancements:
Add visual aids such as a timeline of AMS evolution or a comparative table showing traditional vs. smart AMS features.

II. Desired State of Technology
Strengths:

  • Describes robust goals for the system, including predictive analytics, IoT integration, mobile accessibility, and security.
  • Introduces forward-thinking features like automated emergency protocols and energy monitoring, adding depth to proposed functionality.

Suggestions for Improvement:

  • Group features thematically (e.g., Tenant-Facing Features, Manager Tools, Security & Sustainability Modules).
  • Mention user interface design principles or accessibility standards to support “outstanding user experience.”
  • Clarify whether the features (e.g., smart locks, dashboards) are conceptual or intended for actual implementation within Casa Oro.

Recommended Enhancements:
Include a table summarizing proposed modules, their target users, and expected impact on operations.

III. Statement of the Problem
Strengths:

  • Identifies key operational challenges: fragmented communication, absence of automation, weak security, and lack of community engagement.
  • Uses concise bullet points to present pain points clearly and directly.

Suggestions for Improvement:

  • Reframe each statement into formal research questions for use in instrument development and validation.
    • Example: “What are the effects of predictive maintenance features on tenant satisfaction and cost-efficiency in property management?”
  • Add a brief transitional paragraph that links problems to current gaps in technological capability and user experience.

Recommended Enhancements:
Connect problem areas with existing literature for better alignment with Chapter II.

IV. Objectives of the Study
Strengths:

  • General and specific objectives are well-aligned with identified problems and proposed system components.
  • Clearly targets automation, analytics, financial tracking, and communication management.

Suggestions for Improvement:

  • Add clarity to technical terms (e.g., "predictive maintenance features") to ensure comprehension by non-specialist readers.
  • Link each specific objective to measurable performance indicators (e.g., reduced processing time, improved tenant response rates).

Recommended Enhancements:
Consider reorganizing specific objectives into thematic categories: System Automation, Maintenance Optimization, Communication Enhancement, Financial Management, and Data Analytics.

V. Theoretical Framework
Strengths:

  • Employs Task-Technology Fit (TTF) Model—a strong and well-established choice for evaluating system adoption and performance.
  • Aligns user needs with technological features through structured categories: Utilization, Performance Impact, Task Characteristics, Technology Characteristics.

Suggestions for Improvement:

  • Clarify sources used in table elements (e.g., how “Fast response times” are defined or measured).
  • Supplement TTF with a secondary framework such as Technology Acceptance Model (TAM) or Information Systems Success Model for triangulated analysis.

Recommended Enhancements:
Include a paragraph justifying the choice of TTF and explaining its relevance to property management systems in the Philippine context.

VI. Conceptual Framework
Strengths:

  • Framework components are clearly labeled and logically structured (Log-in → Dashboard → Payment → Maintenance → Notifications → Admin → Database → Server).
  • Provides concise definitions of each module with user-centered narrative flow.

Suggestions for Improvement:

  • Consider revising the figure to show directional flow or layered architecture (e.g., frontend, backend, and database interactions).
  • Distinguish user roles visually in the diagram (e.g., Tenant, Manager, Administrator).

Recommended Enhancements:
Present framework as a system interaction diagram (e.g., UML use-case or sequence diagram) for technical depth.

VII. Operational Definition of Terms
Strengths:

  • Definitions are technically accurate and relevant to the system’s architecture.
  • Focuses on platform-critical concepts such as web servers, notifications, and databases.

Suggestions for Improvement:

  • Include citations for technical definitions using APA style (e.g., IEEE or ACM glossary references).
  • Add definitions for other essential terms used in later chapters (e.g., “Predictive Analytics,” “Smart Security,” “Cloud-Based System”).

Recommended Enhancements:
Turn this section into a glossary appendix with expanded definitions and citations.

VIII. Scope and Limitations
Strengths:

  • Clearly defines scope in terms of system modules, users, and deployment location.
  • Details limitations including geographic confinement, user accessibility challenges, connectivity dependencies, and legal exclusions.

Suggestions for Improvement:

  • Organize content into two-column format for easy scanning:
    • Scope: User roles, features, deployment platform, data architecture.
    • Limitations: Internet dependency, user digital literacy, legal omissions.

Recommended Enhancements:
Add a “Delimitations” paragraph explaining what the study intentionally excludes (e.g., multi-property scaling, mobile app development).

IX. Significance of the Study
Strengths:

  • Covers diverse stakeholders (Tenants, Property Managers, Maintenance Staff, Administrators, Future Developers).
  • Highlights system impact on efficiency, communication, and strategic planning.

Suggestions for Improvement:

  • Strengthen academic tone by rephrasing expressions like “save time and reduce the hassle” to “streamline tenant interaction and administrative burden.”
  • Discuss alignment with sustainability goals or institutional mandates.

Recommended Enhancements:
Include links to broader societal goals (e.g., smart urban housing, digital public services) to reinforce long-term relevance.
