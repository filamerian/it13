Proposed System Analysis

Spinzone Laundry Management and Booking System

Overall Evaluation

This is a well-structured and acceptable BSIT capstone project with a clear real-world problem, defined users, and a practical system workflow.

The revised scope is now properly aligned with what can realistically be implemented within a capstone timeframe.

The system is no longer treated as a full business ERP, but as a focused service management and tracking system, which makes it appropriate for BSIT standards.

Validation of the Problem

The problem is valid and observable in real laundry operations:

  • manual logbook recording leads to errors and missing entries
  • customers lack visibility on laundry status
  • staff struggle to manage multiple service requests efficiently
  • there is no centralized system for tracking transactions and history

These issues are practical, recurring, and measurable, which makes them suitable for a capstone study.

Evaluation of System Scope

The revised scope is now appropriate and manageable. It focuses on the core workflow:

  • service request submission
  • centralized record management
  • laundry status tracking
  • notifications
  • basic reporting

This is a correct capstone-level system size. It avoids unnecessary complexity such as full logistics tracking, advanced analytics, or inventory systems that are not clearly defined.

BSIT Alignment

The project strongly aligns with BSIT competencies, including:

  • web-based system development
  • database design and management
  • CRUD operations
  • user authentication and role management
  • notification systems
  • transaction processing and reporting

This is consistent with CHED expectations for BSIT capstone outputs.

Technical Feasibility

The system is fully implementable using standard technologies such as:

  • PHP or Laravel
  • MySQL database
  • basic notification services (email or SMS API)
  • web dashboard for staff
  • mobile-responsive interface for customers

No part of the system requires advanced or risky technologies.

Novelty Assessment

The innovation is not in complex technology but in:

  • digitizing a manual laundry workflow
  • improving customer transparency through real-time status tracking
  • centralizing service operations into a single system

This level of novelty is acceptable for BSIT capstone standards.

Risks and Limitations

The system is low-risk. However, the group should ensure:

  • clear workflow definition for laundry status updates
  • proper role separation between staff and customers
  • consistent database design for transactions and history

Problem Statement

  1. Laundry businesses still rely on manual logbooks to record customer service requests, which often leads to missing or inaccurate records.
  2. There is no centralized system for managing and organizing laundry service requests from customers.
  3. Customers do not receive real-time updates on the status of their laundry, leading to frequent follow-ups and inconvenience.
  4. Staff experience difficulty in tracking and managing multiple laundry orders efficiently using manual processes.
  5. Laundry operations lack a simple system that can store, monitor, and retrieve transaction history for reporting purposes.

Objectives

General Objective

To design and develop a web-based laundry management system that automates service tracking, status updates, and notifications for Spinzone Laundry.

Specific Objectives

  1. To develop a system that allows customers to submit laundry service requests.
  2. To implement a centralized system for managing all laundry transactions.
  3. To provide real-time laundry status tracking for customers.
  4. To develop a notification system that updates customers about their laundry progress.
  5. To generate simple reports for monitoring laundry operations and sales.

Final Recommendation

This project is acceptable and practical for a BSIT capstone. It should be approved with minor revisions focusing on scope reduction.

The main improvement needed is simplification. The system should focus only on the essential workflow: service request, processing, status update, and notification.

If the scope is controlled, this can become a successful and fully implementable capstone project.
